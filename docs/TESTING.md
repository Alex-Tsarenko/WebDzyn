# Тестирование WebDzyn сервера

## Методы проверки работоспособности

### 1. Автоматический тест-скрипт

Запустите автоматический тест всех функций сервера:

```bash
# Сначала запустите сервер в одном терминале
npm start

# В другом терминале запустите тесты
node server/test-server.js
```

Скрипт проверит:
- ✅ REST API endpoints (`/api/health`, `/api/rooms`)
- ✅ WebSocket подключение
- ✅ Создание комнаты
- ✅ Присоединение к комнате
- ✅ Обмен WebRTC сигналами (offer, answer, ICE candidates)

### 2. Ручная проверка через curl

#### Проверка health endpoint:
```bash
curl http://localhost:3000/api/health
```

Ожидаемый ответ:
```json
{
  "status": "ok",
  "timestamp": 1234567890,
  "rooms": 0,
  "users": 0
}
```

#### Проверка списка комнат:
```bash
curl http://localhost:3000/api/rooms
```

Ожидаемый ответ:
```json
{
  "rooms": []
}
```

### 3. Проверка через браузер

1. Откройте в браузере: `http://localhost:3000/api/health`
2. Должен отобразиться JSON с информацией о сервере

### 4. Проверка WebSocket через консоль браузера

Откройте консоль разработчика (F12) и выполните:

```javascript
// Подключение к серверу
const socket = io('http://localhost:3000');

// Слушаем событие подключения
socket.on('connect', () => {
  console.log('Подключено! Socket ID:', socket.id);
  
  // Создаем комнату
  socket.emit('create-room', (response) => {
    console.log('Комната создана:', response);
  });
});

// Слушаем ошибки
socket.on('connect_error', (error) => {
  console.error('Ошибка подключения:', error);
});
```

### 5. Проверка логов сервера

При запуске сервера вы должны видеть:

```
🚀 WebDzyn Server running on http://localhost:3000
📡 WebSocket server ready for connections
```

При подключении клиента:
```
[2026-03-23T20:57:00.000Z] [INFO] New client connected { socketId: 'abc123' }
```

### 6. Тестирование с двумя браузерами

1. Откройте два окна браузера
2. В первом окне создайте комнату
3. Скопируйте ID комнаты
4. Во втором окне присоединитесь к комнате по ID
5. Проверьте логи сервера на наличие событий `user-joined`


## Устранение неполадок

### Сервер не запускается

**Проблема**: `Error: listen EADDRINUSE: address already in use`

**Решение**: Порт 3000 занят, измените порт в `.env` или остановите процесс:
```bash
# Найти процесс на порту 3000
lsof -i :3000

# Остановить процесс
kill -9 <PID>
```

### WebSocket не подключается

**Проблема**: `connect_error`

**Решение**:
1. Проверьте, что сервер запущен
2. Проверьте CORS настройки в `server/config.js`
3. Убедитесь, что используете правильный URL

### Комната не создается

**Проблема**: Callback не вызывается

**Решение**:
1. Проверьте логи сервера
2. Убедитесь, что WebSocket подключен
3. Проверьте формат запроса

## Мониторинг в реальном времени

Для просмотра активных комнат и пользователей:

```bash
# Постоянный мониторинг
watch -n 1 'curl -s http://localhost:3000/api/health | jq'
```

Или используйте инструменты:
- **Postman** - для тестирования REST API
- **Socket.io Client Tool** - для тестирования WebSocket
- **Browser DevTools** - Network tab для WebSocket соединений

## Нагрузочное тестирование

Для проверки производительности можно использовать:

```bash
# Установить artillery
npm install -g artillery

# Создать конфиг artillery.yml и запустить
artillery quick --count 10 --num 50 http://localhost:3000/api/health
```
